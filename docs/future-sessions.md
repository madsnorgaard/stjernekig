# Future sessions — parking lot

Phase 1 (sessions 01–06) is laid out in [course-outline.md](course-outline.md). Anything that doesn't fit in those 30-min slots lives here.

## Phase 2 candidates

Pick the next two topics based on what colleagues actually ask for at the close of Phase 1.

- **Tests · PHPUnit + Drupal test traits.** Have Claude write tests for the queue worker, run them, watch them fail, fix the code (or the test).
- **D7 → D10 migration.** Use `/drupal-migration` on a CSV of historical APODs (pretend it's a legacy site). Walk through Migrate Plus YAML scaffolding live.
- **Drupal 10 → 11 upgrade.** Use `upgrade_status` and `/drupal-expert` to plan a major version upgrade with a real changelog.
- **A real client repo.** Take cbs-dk or ejerlederen-dk into a session and apply the Phase 1 patterns to actual paid work. Higher stakes; better learning.
- **MCP servers.** Build a small MCP server that exposes the NASA APOD API and consume it via Claude Code. Bridges the Claude-Code series into Claude SDK territory.
- **`/babysit-prs`.** Set up Claude to monitor PRs across the team and post status to Slack. Pairs naturally with Session 05.
- **Visual debugging with Playwright.** Use the visual analyzer skills to test the gallery rendering across breakpoints.

## Format ideas

- **Pair-programming sessions.** Cap at 4 people, 60 min, work on a real bug in cbs-dk together with Claude.
- **Hackathon-style.** Half-day. Pick a tiny green-field idea per team, ship it with Claude. Demo at the end.
- **External speaker.** Invite someone from another agency or Drupal contrib team. *"Sådan bruger I Claude. Sådan bruger vi Claude. Hvad lærer vi af hinanden?"*

## Stuff that probably doesn't belong here

- **A "Claude vs. Cursor" comparison session.** Tempting but unproductive — both tools, different shapes. Better to teach Claude deeply than to half-explain two tools.
- **Generic AI ethics talk.** Not bad, but not this series. That's a separate org-wide thing.
